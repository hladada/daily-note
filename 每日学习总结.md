# 我的第一天 Markdown 笔记
## 一、Markdown 标题层级学习
### 1.1 六级标题演示
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
> 小知识点：# 和文字之间**必须空一格**，否则不会识别为标题，字号从一级到六级依次变小。

## 二、段落与换行规则
### 2.1 分段（硬分段）
写 Markdown 段落时，两段文字中间**空一行**才会分成两个独立段落。

今天正式入门 Markdown，它语法简单易上手，专门用来快速排版纯文本，不用复杂的办公软件。

日常写笔记、文档、README 都会用到，学会基础格式就能做出层次清晰的笔记。

### 2.2 段落内软换行
在文字末尾敲**两个空格再回车**，只换行不分段，属于同一段落：
整理笔记有两个核心好处：  方便后续复习查阅
降低后期修改文档的成本

## 三、文字强调格式（斜体、粗体、粗斜体）
### 3.1 基础强调符号
1. 单星号斜体：*文字倾斜展示*
2. 双星号粗体：**文字加粗突出**
3. 三星号粗斜体：***同时加粗倾斜***

下划线写法效果一致：
_斜体文字_
__粗体文字__
___粗斜体文字___

### 3.2 嵌套混合强调
粗体内嵌斜体：**这份笔记里*重点标记*的内容一定要记住**
斜体内嵌粗体：*所有语法**符号必须成对闭合**，否则格式会错乱*

## 四、转义字符：原样显示格式符号
如果想直接打印 * 和 #，需要在符号前加反斜杠 \ 转义：
1. 数学乘号 \* 不会变成斜体标记
2. \# 单独书写不会识别为一级标题

### 今日学习总结
***核心要点汇总***：
1. 标题用 # 分级，空格不能少；
2. 空行分段，两空格回车实现段内换行；
3. * / ** / *** 控制文字强调，支持互相嵌套；
4. \ 反斜杠可转义，原样输出格式标记 \*。

# 理解版本控制的概念
## 什么是版本控制
### 版本控制是一套记录文件修改历史的工具，能保存每一次改动快照，实现：
*多人协作修改不冲突，清晰查看谁、什么时候改了什么内容  
Git 属于分布式版本控制：每个人电脑都有完整仓库副本，不依赖网络也能本地记录版本。*

### 三个核心区域
#### 工作区：你电脑文件夹里看得见的文件（day-01.md 正在编辑的文件）
#### 暂存区（Stage）：临时存放待提交改动的缓冲区，手动把文件放进来
#### 本地仓库：硬盘里隐藏的 .git 文件夹，永久保存每一次版本快照

#### 流转逻辑：工作区修改 → git add 存入暂存区 → git commit 存入本地仓库

绿色文件：已加入暂存区，等待 commit*
