# git 基础

## 1.仓库初始化

- 本地新建文件夹，执行：
- git init
- 还有一种方法是直接克隆远程的仓库，执行：
- git clone 仓库地址

## 2.两个命令的对比对象

* git diff 比较的是“你正在编辑的代码”和“暂存区暂存的代码”即改了代码但还没 git add
* git diff --stage 比较的是“暂存的代码”和“最后一次提交到仓库的代码”即已将代码 git add 但还没git commit

### 比较两个具体提交的差异

* 代码格式：git diff 旧ID 新ID；

查看提交历史找到你要的历史版本

* git log --oneline 

这里我所选用的旧ID：3209df 新ID：9eccfe6

* git diff 3209df3 9eccfe6

如果你只要对比某个具体文件的提交差异

* git diff 3209df3 9eccfe6 day-02.md

## 3. 撤销暂存区暂存的内容

方法一：

* 撤销指定文件的暂存：
git restore --staged day-02.md

* 撤销所有文件的暂存：
git restore --staged .

 方法二：

* 撤销指定文件的暂存：
git reset HEAD day-02.md
* 撤销所有文件的暂存
git reset HEAD

## 4.撤销本地仓库内容

1. 只撤销工作区的内容，退回与暂存区一致

* 恢复单个文件：
git restore day-2.md

* 丢弃所有工作区修改：
git restore .

1. 撤销本地提交即已经提交但还没pull到远程仓库

* 撤销最近1次提交，修改保留在工作区
git reset --mixed HEAD~1
git reset HEAD~1

## 5.撤销远程仓库内容

* 原理：新增一条反向提交抵消错误提交的改动，不会删除历史、不会改写远程提交记录，不受分支保护规则拦截

同步远程仓库的最新代码

* git pull origin main

撤销指定的错误提交，替换为你要撤销的错误提交ID

* git revert ce17dab53ed93a31c4526fcaf251986e8913f260（此代码为要撤销的ID）

正常push到远程仓库

* git push origin main

### 补充一条强制退回的方案

* 原理：本地先回退到正确版本，再强制覆盖远程历史，彻底删除错误提交。

1. 本地硬重置到目标正确版本

* git reset --hard 9eccfe6

1. 强制推送到远程仓库

* 相对安全：会先检查远程是否有他人新提交，避免误删
git push origin main --force-with-lease

* 无条件强制推送（仅确认远程无他人提交时使用）
git push origin main -f

* ps：多人协作无法使用会导致其他人无法正常推送；如果远程仓库开启了分支保护规则，强制推送会被直接拒绝，需要先在 GitHub 仓库设置中开启允许“强制推送”
