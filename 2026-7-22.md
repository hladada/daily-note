# 今日学习总结

## 1.初始化仓库

### 1.1 在项目文件夹执行

- git init
- git remote add origin <git@xxx.git>

## 2.提交内容到本地仓库

### 追踪所有的变更的文件

- git add .

### 提交说明

- git commit -m "初始提交"

## 3.推送本地仓库至远程仓库

## 推送main分支，建立关联

- git push -u origin main

## 4.创建新分支：feat-001

- git branch feat-001

# 5.切换到分支：feat-001

- git checkout feat-001

# 6. 在 feat-001 提交内容并推送到本地仓库

## 修改文件后

- git add .
- git commit -m "feat: 001新增功能"

## 7.将feat-001 推送到远程仓库

- git push -u origin feat-001

## 8.发起 PR，合并 feat-001 内容到 origin/main

- git checkout main
- git pull origin main

## 9. 在本地删除 feat-001，并同步仓库

### 先切回main分支，不能在待删除分支内执行

- git checkout main

### 删除本地分支

- git branch -d feat-001

### 删除远程分支

- git push origin --delete feat-001

## 10. 回退 main 分支上 feat-001 修改的内容

`1.revert：留历史记录，适合公共远程分支`

- git revert <feat合并提交的commit-id>
- git push origin main

`2.reset：仅适合个人仓库，强制回退`

- git reset --hard <合并前的commit-id>
- git push -f origin main

## 11.创建基于 main 的 worktree

`格式：git worktree add [工作目录路径] [分支]`

- git worktree add ../worktree-main main

## 12.在新的 worktree 的 main 上提交内容并推送到远程仓库

### 进入新建的worktree目录

- cd ../worktree-main

### 修改文件

- git add .
- git commit -m "worktree 中main分支更新"
- git push origin main

## 13. 删除创建的 worktree

### 先执行git命令注销worktree

- git worktree remove ../worktree-main

### 强制删除（有未提交修改时）

- git worktree remove --force ../worktree-main

### 查看所有worktree确认

- git worktree list
