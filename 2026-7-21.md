## 分支：创建、切换、合并、删除全套操作
### 1.创建分支
- git checkout -b feature/login
### 2.切换分支
- get swich main
### 3.合并分支
##### 3.1. 先切到目标分支（main）
- git switch main
##### 3.2. 合并功能分支
- git merge feature/login
### 若出现冲突，手动修改文件后执行 git add . && git commit 完成合并
### 4.删除分支
#### 4.1删除本地已合并完毕的分支
- git branch -d feature/login
#### 4.2 强制删除未合并的分支
- git branch -D feature/login

## HEAD 指针是什么
1. 用最简单的比喻
- 整个 Git 仓库的所有提交，就像一长串排队的照片，每张照片代表一次代码保存记录。
- main、feature 这种分支名：相当于贴在某张照片上的便利贴标签，固定标记一个版本；
- HEAD：就是你手里的光标 / 手指，手指指到哪一张照片，你电脑打开的代码就是对应这个版本。
- 正常状态：HEAD 绑定在分支名上（例：HEAD -> main），切换分支时 HEAD 跟着移动；
- 分离头指针状态（detached HEAD）：直接切到某个 commit 哈希，不绑定任何分支，临时改代码不会保存到分支，操作完务必切回分支。

## rebase 和 merge：作用、适用场景
### 1. git merge
- 将两个分支完整合并，生成一条新的合并提交记录，保留两条分支完整提交历史，历史交叉清晰可见。
### 2. git rebase
- 把当前分支所有提交，移植到目标分支最新节点后面，重写提交历史，消除分叉，形成一条直线干净日志。

## worktree 工作树：概念、场景、命令
1. 什么是 worktree
- 常规 Git 仓库只有一个工作目录，同一时间只能切换一个分支；
- git worktree 可以给同一个仓库创建多个独立工作文件夹，每个文件夹绑定不同分支，多分支代码同时存在、互不干扰。

2. 创建新工作树（绑定指定分支）
- git worktree add ../bug-fix main(在 ../bug-fix 文件夹创建worktree，分支main)

3. 查看所有已创建的工作树列表
- git worktree list

4. 删除无用工作树
- git worktree remove ../bug-fix

5. 清理已手动删除文件夹的无效 worktree 记录
- git worktree prune

6. 给新 worktree 创建全新分支
